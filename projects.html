MuJoCo Sim2real PyTorch Markov Reward Process

Overview

Developed a simulation-driven machine learning model for predicting fall-from-bed risk from a single static patient posture as part of a smart ICU bed system.

Since real-world fall events are rare and ethically difficult to collect at scale, I built a physics-based human–bed simulation environment using MuJoCo to generate large-scale labeled training data. The simulator samples diverse initial in-bed postures and rolls out uncontrolled physical dynamics to determine whether a fall occurs.

Each simulated episode is converted into a 13-keypoint 2D skeleton representation, and fall risk is formulated as a continuous regression problem based on discounted time-to-fall. This formulation allows the model to estimate a risk score from a single static posture without requiring temporal motion sequences.

From a reinforcement learning perspective, the model can be interpreted as learning a value function that predicts the expected discounted return of a fall event under a fixed no-control policy, enabling per-frame fall risk estimation in bedside monitoring systems.

Result

The proposed simulation-based approach demonstrates that meaningful fall-risk signals can be inferred from a single static in-bed posture without requiring temporal motion sequences. Through large-scale physics-based simulation and supervised learning, the model successfully learned posture-dependent risk patterns and produced stable fall-risk estimates across diverse patient configurations.

The study shows the feasibility of using synthetic simulation data to train predictive safety models in healthcare scenarios where collecting real-world fall data is difficult due to safety and ethical constraints. These results suggest that simulation-driven learning could serve as a practical foundation for developing proactive bedside monitoring systems capable of estimating fall risk before hazardous events occur.

My Role

  • Designed and implemented the physics-based bed–human simulation pipeline using MuJoCo.
  • Developed the data generation and labeling framework for large-scale fall-risk datasets.
  • Engineered the skeleton feature extraction and normalization pipeline from simulator states.
  • Formulated fall risk prediction as a continuous value prediction problem based on discounted time-to-fall.
  • Conducted experiments on dataset construction, sampling strategies, and scaling behavior to analyze model generalization.
Computer Vision Multitask Model PyTorch Dynamic Vision Sensor

Overview

Developed a vision-based patient monitoring model for a smart ICU bed system that automatically responds to patient status detected from visual sensor inputs. The overall goal of the project is to improve patient safety by enabling the bed to react intelligently to high-risk situations, such as prolonged static postures related to pressure ulcer risk.

To support real-world deployment in hospital environments, I adopted an RGB–DVS multimodal approach. The DVS sensor was introduced as a privacy-preserving modality because it captures motion events without exposing detailed identifiable appearance information. In addition, I designed a multitask learning model that simultaneously performs RGB anonymization and patient posture classification, allowing the system to preserve visual privacy while maintaining monitoring performance.

The developed model is intended to serve as a perception module for the smart bed, providing reliable posture-related signals that can later be used to trigger bed-level actions such as air mattress adjustment or patient repositioning.

Result

Built a privacy-aware patient monitoring pipeline using RGB and DVS sensors for smart ICU bed applications. The proposed model demonstrated the feasibility of combining posture classification and visual anonymization within a unified framework, while also supporting practical deployment in medical environments where privacy is critical.

In addition to model development, the system was prepared for embedded deployment, enabling real-time inference on an embedded board for interaction with the smart bed platform. This established the technical foundation for integrating AI-based patient monitoring with bed control in future ICU-bed applications.

My Role

  • Designed and implemented the RGB–DVS-based patient posture classification model.
  • Developed a multitask learning architecture for simultaneous RGB anonymization and posture classification.
  • Processed and synchronized multimodal sensor data from RGB and DVS cameras.
  • Optimized and deployed the model on an embedded board for real-time inference.
Graduation Project Python jax Multimodal ViT

Overview

Developed an audio–visual multimodal cry detection model for a smart infant bed system designed to automatically respond to infant states (e.g., crying, environmental adjustment).

The project addresses the limitation of audio-only cry detection systems, which are highly sensitive to environmental noise and often produce false alarms. To improve robustness, a multimodal architecture was designed by combining a pretrained Vision Transformer (ViT)-based video encoder with an audio feature extraction model. By leveraging both facial expressions and body movements from video alongside acoustic features from crying sounds, the system performs binary classification of infant crying vs. non-crying states with improved reliability in real-world environments.

Result

The multimodal fusion approach significantly reduced false positives caused by external noise compared to audio-only detection systems. By integrating visual context with acoustic signals, the model achieved more stable cry detection performance in noisy environments. The developed model demonstrated improved robustness for deployment in smart infant care devices, enabling more reliable automated responses within the smart baby bed system.

My Role

  • Designed and implemented the audio–visual multimodal architecture integrating a pretrained ViT-based video encoder with an audio feature model using jax.
  • Processed and synchronized video and audio datasets to build a multimodal training pipeline.

LLM Prompt Engineering Socket Communication Ollama

Overview

Developed a voice-interactive chatbot for depression screening based on the PHQ-9 questionnaire. The system aims to simulate a natural conversational interaction while conducting structured mental health assessments.

To achieve this, a Large Language Model (LLM) was prompt-tuned to follow a rule-based diagnostic workflow derived from the PHQ-9 clinical questionnaire. The chatbot guides users through nine screening questions in a conversational manner while maintaining the predefined diagnostic logic.

The system integrates speech-to-text (STT) and text-to-speech (TTS) components to enable natural voice interaction. Due to the computational requirements of the LLM, the architecture separates the client and server: the client extracts speech input and sends it to the server via Socket communication, where the LLM generates responses using Ollama. The generated responses are then returned to the client and converted to speech for playback.

Result

The developed system successfully demonstrates an end-to-end voice-based depression screening pipeline, combining speech processing, LLM-based dialogue generation, and structured diagnostic scoring.

The chatbot conducts the full PHQ-9 assessment through conversational interaction and automatically calculates the final score once all nine questions are completed, providing a simple demonstration of how LLM-based conversational systems can support structured mental health screening tasks.

My Role

  • Designed and implemented the entire system architecture, including client–server communication and voice interaction pipeline.
  • Prompt-engineered the LLM to follow PHQ-9 diagnostic rules while maintaining conversational dialogue with users.
  • Implemented Socket-based communication between the local client and LLM inference server.
  • Integrated Speech-to-Text (STT) and Text-to-Speech (TTS) modules to enable real-time voice interaction.
  • Deployed and managed the LLM using Ollama to support local inference of large models.
  • Implemented the PHQ-9 scoring logic to automatically compute and reveal the screening result after the questionnaire is completed.
